# Form Validation UX Strategy

## Presentation Rules
1. **Real-Time Validation**: Rely on Zod schema checks tied to `react-hook-form` `onChange` or `onBlur` listeners. 
2. **Accessible Messaging**: Errors mapping to form controls using `messages/*` strictly populated inside UI form components: `<FormMessage>`.
3. **Step Guards (Wizard Components)**: Ensure state machines blocking navigation cleanly render failure summaries *before* the user advances a virtual page. 
   - Forward jumps (URL spoofing) are strictly trapped by `useEffect` interceptors reverting the state back to the lowest valid step index.
4. **Scroll & Focus**: Utilize custom hooks (`useFormScroll` or similar native DOM methods inside layout containers) targeting `alert` nodes generated by schema rejections to drag the user's viewport safely upwards.

## Constraints
Developers must never inject `.optional()` on Zod validations simply to bypass UI debugging errors. Proper `{ defaultValues: { ... } }` must satisfy the core type rules prior to first-pass renders.
